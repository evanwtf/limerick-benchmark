## Key Findings

**1. Small models outperformed large ones.**  
All three gemma4 variants under 5B parameters passed quickly and cleanly. The 26B gemma4 failed. Among Qwen models, only the tiny 9B and the specialized 35B-coding variant passed.

**2. Larger models exhibit planning paralysis.**  
The qwen3.6 and most qwen3.5 large models spent their entire time budget designing rhyme dictionaries they never wrote to disk. The failure mode was uniform: workspace unchanged at 180-second timeout.

**3. "Coding" specialization helped for large Qwen models.**  
`qwen3.5:35b-a3b-coding-mxfp8` passed while `qwen3.5:35b-a3b` failed with the same 180s timeout. The coding variant also produced the highest-quality implementation of any passing run.

**4. Fastest passing model: gemma4:e2b at 13 seconds.**  
The tiny 2B model shipped a working app nearly 6x faster than the next-best passing model (gemma4:e2b-mlx-bf16 at 23.9s) and produced correct output with zero overthinking.

**5. Best code quality: qwen3.5:35b-a3b-coding-mxfp8.**  
The only model to implement actual generative logic (randomized rhyme-word selection from phonetic groups) rather than a static list, while still passing the evaluation.
